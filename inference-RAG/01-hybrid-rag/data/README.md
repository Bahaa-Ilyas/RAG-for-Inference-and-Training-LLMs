# Data Guide

This folder contains the normalized data used by the Hybrid RAG cookbook.

The raw-looking fixtures live in `../examples/` and `../docs/`. The retrieval-ready rows live in `corpus.jsonl`.

## Pipeline Responsibility

This data folder is part of the end-to-end RAG responsibility model. Ingestion, chunking strategy, metadata preservation, retrieval evaluation, and response quality are connected. If a source row loses its page number, section, URL, version, ACL, or entity metadata here, the answer layer cannot reliably recover it later.

## Files

| File | Purpose |
|---|---|
| `corpus.jsonl` | Mixed-source retrieval corpus. One JSON object per document or document element. |
| `queries.jsonl` | Example user queries for local evaluation and demos. |
| `qrels.jsonl` | Query relevance labels. Each query maps to one or more expected source IDs. |
| `local_index.json` | Generated by `2-build-index.py`; ignored by git and safe to delete. |

## Corpus Schema

Each row in `corpus.jsonl` follows this shape:

```json
{
  "id": "stable_source_or_chunk_id",
  "title": "Human-readable title",
  "source_type": "pdf | scanned_pdf | webpage | table | tool | contract_pdf_text | incident_report | security_control | policy",
  "path": "relative/path/to/source_fixture",
  "text": "retrieval text",
  "metadata": {
    "page": 12,
    "section": "source section",
    "version": "2026.04",
    "is_current": true,
    "acl": ["public"],
    "entities": ["Vendor", "Security Review"]
  }
}
```

## Why These Are Harder Than Toy Documents

These enterprise fixtures deliberately mix exact strings and semantic intent:

- `MSA-2026`, `Section 8.2`, and `CC6.2` are exact identifiers that BM25 should catch.
- Questions such as "who owns remediation" or "how long are logs kept" require semantic matching.
- Access-control and procurement examples contain thresholds, roles, and workflow constraints.
- Web retention data includes freshness/version metadata that production systems should preserve.

Hybrid RAG is useful because it can retrieve both exact identifiers and paraphrased concepts before rank fusion.
